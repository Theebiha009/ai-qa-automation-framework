#### 1. Functional Scenarios

##### Test Case 1: Successful Login
- **API Endpoint**: `POST /api/login`
- **Request Payload**:
  ```json
  {
    "username": "validUser",
    "password": "validPassword"
  }
  ```
- **Expected Response**:
  - Status Code: `200 OK`
  - Response Body:
    ```json
    {
      "message": "Login successful",
      "token": "<JWT or session token>"
    }
    ```

##### Test Case 2: Invalid Password
- **API Endpoint**: `POST /api/login`
- **Request Payload**:
  ```json
  {
    "username": "validUser",
    "password": "invalidPassword"
  }
  ```
- **Expected Response**:
  - Status Code: `401 Unauthorized`
  - Response Body:
    ```json
    {
      "error": "Invalid username or password"
    }
    ```

##### Test Case 3: Non-existing Username
- **API Endpoint**: `POST /api/login`
- **Request Payload**:
  ```json
  {
    "username": "nonExistingUser",

  }
  ```
- **Expected Response**:
  - Status Code: `401 Unauthorized`
  - Response Body:
    ```json
    {
      "error": "Invalid username or password"
    }
    ```

#### 2. Negative Cases

##### Test Case 4: Lock Account After 5 Failed Attempts
- **API Endpoint**: `POST /api/login`
- **Scenario**: Attempt to log in with incorrect password 5 times
- **Request Payload** (5 consecutive requests):
  ```json
  {
    "username": "validUser",
    "password": "invalid"
  }
  ```
- **Expected Response** (after 5 attempts):
  - Status Code: `423 Locked`
  - Response Body:
    ```json
    {
      "error": "Account locked due to too many failed login attempts. Please try again later."
    }
    ```

##### Test Case 5: Attempt Login After Account is Locked
