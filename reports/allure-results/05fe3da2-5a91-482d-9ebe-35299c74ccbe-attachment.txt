### Positive Cases

---

**Test Case ID:** TC001  
**Description:** Successful login with valid credentials.  
**Steps:**  
1. Send POST request to `/api/login` with valid JSON body:  
   ```json
   {
       "username": "validUser",
       "password": "validPassword"
   }
   ```  
2. Check the HTTP response status code.  
3. Validate the response body for success message and token.  

**Expected Result:**  
- HTTP Status Code: 200  
- Response Body:  
  ```json
  {
      "message": "Login successful",
      "token": "generatedToken"
  }
  ```

### Edge Cases

---

**Test Case ID:** TC006  
**Description:** Attempt to login with extremely long username and password.  
**Steps:**  
1. Send POST request to `/api/login` with JSON body:  
   ```json
   {
       "username": "a".repeat(256),
       "password": "p".repeat(256)
   }
   ```  
2. Check the HTTP response status code.  

**Expected Result:**  
- HTTP Status Code: 400 (or appropriate error code based on validation)  
- Response Body:  
  ```json
  {
      "error": "Username and password must be less than 255 characters"
  }
  ```

---

**Test Case ID:** TC007

   ```json

       "username": "' OR '1'='1",
       "password": "anyPassword"
   }
   ```  
2. Check the HTTP response status code.  

**Expected Result:**  
- HTTP Status Code: 401 (or appropriate error code)  
- Response Body:  
  ```json
  {
      "error": "Invalid credentials"
  }
  ```

---

**Test Case ID:** TC008  
**Description:** Attempt to login with special characters in username and password.

**Expected Result:**

  ```json
      "message": "Login successful", 
      "token": "generatedToken" 
  }
  ```  
  or  
  ```json
  {
      "error": "Invalid characters in username or password"
  }
  ```
